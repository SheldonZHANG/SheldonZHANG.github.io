17.0.5 设置忽略文件
一般我们总会有些文件无需纳入 Git 的管理，也不希望它们总出现在未跟踪文件列表。在这种情况下，我们可
以创建一个名为.gitignore 的文件，列出要忽略的文件模式。来看一个实际的例子：
cat .gitignore
*.[oa]
*~
# no .a files
*.a
# but do track lib.a, even though you're ignoring .a files above
!lib.a
# only ignore the TODO file in the current directory, not subdir/TODO
/TODO
# ignore all files in the build/ directory
build/
# ignore doc/notes.txt, but not doc/server/arch.txt
doc/*.txt
# ignore all .pdf files in the doc/ directory
doc/**/*.pdf

17.0.7 Di
• 在 git 提交环节，存在三大部分：working tree, index le, commit
– working tree：就是你所工作在的目录，每当你在代码中进行了修改，working tree 的状态就改变了。
– index le：是索引文件，它是连接 working tree 和 commit 的桥梁，每当我们使用 git-add 命令来登记
后，index le 的内容就改变了，此时 index le 就和 working tree 同步了。
– commit：是最后的阶段，只有 commit 了，我们的代码才真正进入了 git 仓库。我们使用 git-commit 就
是将 index le 里的内容提交到 commit 中。

– git di –cached：是查看 index le 与 commit 的差别的。
– git di HEAD：是查看 working tree 和 commit 的差别的。（你一定没有忘记，HEAD 代表的是最近的
一次 commit 的信息）

• git rm: 从暂存区和工作目录 (也即磁盘上) 一并删除某个文件; 如果删除之前修改过并且已经放到暂存区域
的话，则必须要用强制删除选项 -f（译注：即 force 的首字母）
• 先直接在磁盘上利用 “rm your_le_name” 命令删除，然后用 git status 即可以发现已经从工作区中删除, 最
后用 git rm 最终删除; “‘ $ rm PROJECTS.md $ git status On branch master Your branch is up-to-date with
‘origin/master’. Changes not staged for commit: (use “git add/rm …” to update what will be committed)
(use “git checkout – …” to discard changes in working directory)
deleted: PROJECTS.md
no changes added to commit (use “git add” and/or “git commit -a”)
+ git rm --cached filename: 表示只从暂存区删除而不从磁盘工作目录删除;
+ 通配符: git rm \*~ 表示删除以 ~ 结尾的所有文件。
