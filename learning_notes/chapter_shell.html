14.3 Robust code style
1. 使用 set − u 或全称 set − onounset 来确保变量如果没有声明就直接使用的情况下，程序报错推出
2. 使用 set − e 或全称 set − oerrexit 来确保程序遇到任何返回状态非 0 的，就直接推出。不过使用 set − e
要克制，因为该命令会导致很多其它命令的行为发生很大的变化，出现不可预期的错误。因此要少用，尽量
用在小的脚本里, 或一段子代码段中，比如
^^I^^I^^Iset -e
^^I^^I^^Icommand1
^^I^^I^^Icommand2
^^I^^I^^Iset +e
3. 如果你对一些命令的返回状态不关心，可以使用如下方式
^^I^^I^^Iset +e
^^I^^I^^Icommand1
^^I^^I^^Icommand2
^^I^^I^^Iset -e
4. 检查代码的返回状态, 可以使用如下简便的方式
^^I^^I^^Iret=$(func1) || { echo "command failed"; exit 1; }
^^I^^I^^Iif [[ ! $(func1) ]]; then
^^I^^I^^I^^Iecho "command failed"
^^I^^I^^I^^Iexit 1;
^^I^^I^^Ifi
5. 字符串变量永远使用”” 来包裹
^^I^^I^^Iif [[ "$filename" == "hello.txt" ]]; then
^^I^^I^^I^^Icommand
^^I^^I^^Ifi
^^I^^I^^I# instead of
^^I^^I^^Iif [[ $filename == "hello.txt" ]]; then
^^I^^I^^I^^Icommand
^^I^^I^^Ifi
6. 参数内容用”$@”, 而不是 $@
^^I^^I^^Ifunction foo() {
^^I^^I^^I^^Ifor i in "$@"; do

# to print elements from index to the end
echo ${arr[@]:begin_index}

# to print elements in range
